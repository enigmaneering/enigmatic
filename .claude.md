## Repository Structure

### Core Directories
- `soul/core/mind/` - Main GPU device abstraction layer
  - `transpile/` - Shader transpilation (GLSL ↔ HLSL ↔ MSL via SPIRV)
  - `backend_*.go` - Platform-specific backends (Metal, Vulkan, D3D12)
  - `init.go` - Auto-downloads external libraries on first import (runtime)
  - `ensure_external.go` - Standalone tool for pre-build library download (build-time)
- `external/` - **Auto-downloaded from github.com/enigmaneering/external releases**
  - Library directories (`glslang/`, `spirv-tools/`, `spirv-cross/`, `dxc/`) are downloaded on first use
  - Can optionally track `LICENSES/`, `README.md` for convenience
- `onion/` - Development journal entries

### Build System
- **New Two-Repo System**:
  1. `github.com/enigmaneering/external` - Separate repo hosting pre-built binaries
     - GitHub Actions builds for 6 platforms (darwin-arm64, darwin-amd64, linux-arm64, linux-amd64, windows-arm64, windows-amd64)
     - Published to GitHub Releases (not Actions artifacts)
     - Go module at `go/lib` provides `EnsureLibraries()` function
  2. `github.com/enigmaneering/life` (this repo) - Imports external libraries
     - `go.mod` requires `github.com/enigmaneering/external/go/lib v0.0.42`
     - `mind/init.go` calls `external.EnsureLibraries()` on first import
     - Downloads to `life/external/` directory
- Users never need to manually build C++ dependencies

### Key Files
- `Makefile` - Build targets for downloading libraries and running tests
- `mind/ensure_external.go` - Standalone tool called by Makefile to download libraries before build
- `mind/init.go` - Calls `external.EnsureLibraries()` to auto-download toolchain (runtime fallback)
- `transpile/glslang.go`, `spirv_cross.go`, `dxc.go` - CGo wrappers for C++ libraries
- `transpile/build_check.go` - **DEPRECATED** (legacy, no longer auto-runs)
- `transpile/download_prebuilt.go` - **DEPRECATED** (legacy GitHub Actions artifact downloader)

## Development Workflow

### Making Changes
1. Run tests: `cd soul/core/mind && go test ./...`
2. Check for issues before committing
3. Only commit when user explicitly requests it

### What NOT to Commit
- ❌ Files in `external/{glslang,spirv-tools,spirv-cross,dxc}/` - These are auto-downloaded
- ❌ Build artifacts (`*.o`, `*.a`, `build/` directories)
- ❌ Temporary files

### What TO Track
- ✅ Source code in `soul/`
- ✅ Tests and examples
- ✅ `go.mod` and `go.sum` (includes external library version)
- ⚠️ `external/LICENSES/` and `external/README.md` are optional (can be downloaded)

## Architecture Notes

### Observable References Pattern
The GPU device abstraction uses "Observable References" (inspired by quantum observation):
- GPU memory references have no synchronization guarantees
- Like Schrödinger's cat: data state is uncertain until observed
- References are "irrational" - they exist but their state is indeterminate
- See `device/README.md` for full philosophy

### Shader Transpilation
- All shaders go through SPIRV intermediate representation
- Supports GLSL, HLSL, MSL input/output
- Auto-transpiles based on target backend
- Uses glslang, DXC, and SPIRV-Cross under the hood

## Testing Philosophy

Tests use a shared kernel (`suite_shared.go`) that:
- Runs identical compute shader across all backends
- Verifies cross-language transpilation
- Tests basic GPU operations (buffer allocation, kernel execution)

## Common Tasks

### First Time Setup
```bash
# Download external libraries before first build
make
```

### Update Pre-built Binaries
```bash
# Clean and re-download
make clean
make
```

### Run Tests
```bash
# Automatically ensures libraries are downloaded first
make test

# Or manually
cd soul/core/mind
go test ./...
```

### Check External Repo Build Status
Visit: https://git.enigmaneering.org/enigmaneering/external/actions

### Update External Library Version
```bash
# Edit go.mod to change version
go get github.com/enigmaneering/external/go/fetch@go/fetch/v1.0.43
go mod tidy
```
